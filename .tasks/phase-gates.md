# Phase Gate Tracker

이 문서는 `.tasks/plan.md`의 phase entry/exit gate를 실제 작업 루프에서 추적하기 위한 파일이다. 각 phase는 exit gate를 충족하기 전 다음 phase로 넘어가지 않는다.

## Status Legend

- `not-started`: 아직 시작하지 않음
- `in-progress`: 현재 진행 중
- `blocked`: 진행 조건이 충족되지 않음
- `complete`: exit gate와 evidence가 모두 충족됨

## Gate Rules

- Phase 0 이후 새 태스크는 이전 phase exit gate를 확인하고 시작한다.
- Evidence가 `pending` 또는 `none`이면 해당 phase는 완료로 표시하지 않는다.
- Product code 변경은 Phase 1부터 허용한다.
- 설정, 로그, 상태머신 변경은 해당 phase의 TDD 요구사항을 충족해야 한다.
- MVP scope 밖의 iOS/Android/SaaS 구현은 Phase 0 decision record 없이 시작하지 않는다.
